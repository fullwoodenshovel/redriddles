# Redriddles v1.0.3

## What's new
This is stuff important for the end user, basically the technical changes, but summarised and less important changes ignored.
These are the new features:

- Added a save file
- Made export tab


## Technical changes
These are all of the new commits from this release (ignoring repeats).
This may include technical changes, architectural changes, bug fixes, etc:

- Improved workflow
- Removed terminal showing on windows
- Made clicking a shortcut twice return to original state
- Added start of runtime errors
- Added reading save file architecture
- Added save file. If it is invalid, it will display an error message. Shortcuts and previous directories are now stored persistently
- Standardised most buttons
- Made American English parseable
- Made Select Folder tab
- Created a flamegraph
- Updated example code for Node
- Made shortcuts disableable and refactored hex input
- Made it generate the end result (the image from the pixels)
- Made it load image files and process information about them in parallel, including a loading bar
- Made small scale image generation
- Made attempting to save invalid file formates fisplay an error to the user instead of panicking
- Made temperature effect image correctly
- Added sliders
- Added ability to change export settings
- Made multiline text cut off lines that are too long and put them on the next line
- Adding exporting as pixels
- Seperated averaging colour space from distance colour space


## Downloads

### Windows
- **redriddles-v1.0.3-windows-x86_64.zip**
  1. Extract the ZIP
  2. Open the 'redriddles' folder
  3. Double-click redriddles.exe

### Linux
- **redriddles-v1.0.3-linux-x86_64.zip**
  1. Extract the ZIP
  2. Open the 'redriddles' folder in terminal
  3. Run: `./redriddles`

### macOS
- **redriddles-v1.0.3-macos.zip**
  1. Extract the ZIP
  2. Open the 'redriddles' folder
  3. Right-click 'redriddles' and Open (first time only)
  4. Or run in terminal: `./redriddles`
